# Debayering-Filter
Implementation of a Debayering Filter using VHDL and Vivado.
The filter assumes that the pattern of the input image is GBRG (green, blue, red, green) and generates the output image in RGB for each pixel.
The filter can work for any image size via the N parameter (eg for N=1024 we get an input image of 1024x1024 pixels).

The architecture of the filter is comprised of 3 units:

Calculactor Unit:
This is the unit where each pixel of the output image is generated, meaning that the three color components (RGB) are calculated for each pixel. This unit receives a pixel of the input image (containing only one color component), along with the 3x3 neighborhood of pixels needed to generate the output pixel (calculate the missing two color components). The function of this unit is simpler in comparison to the other two, because all the work needed to provide a usable 3x3 window (and the color case input signal) has been done by the other two unit. So this unit doesn't need to worry about the validity of the window and is only responsible for calculating the RGB components and raising a 'valid_out' signal when each output pixel has been generated.

Data Bank Unit:
This the the storage unit of the filter, where the input pixels are kept in order to create the 3x3 window needed for calculations. It consists of 3 FIFO line buffers of size N, the output of each is connected to the first column of a 3x3 block of registers, like in the image below.
<img width="769" height="284" alt="image" src="https://github.com/user-attachments/assets/e76df582-dc5c-4014-84b1-65f8319aa6a4" />
As the pixels are thrown into the filter, they pass through each FIFO and when exiting each FIFO they also enter the 3x3 window. This eventually creates the appropriate window to be used by the calculator. The data bank also needs to know the position of each pixel inside the image, in order to determine whether it's a first-or-last row and/or column pixel. If it an "edge" pixel, the bank will have to make the appropriate positions of the window empty (because those pixels don't exist in the image). This is done by the 'boundaries' signal, which is generated by the control unit.
By the time the first pixel arrives in the bank, it takes 2*N+2 cycles for the first 3x3 window to be generated (N+N cycles to fill the first two FIFOS and then an extra 2 cycles for the pixel to reach the center position of the register window). This unit is essentially a serial-to-parallel-converter.

Control Unit:
This is the unit responsible for generating all the control signals needed by the other 2 units.
Specifically, it generates the valid_in signals for the bank and the calculator along with the initialization signal for it, whenever there is a new image to be filtered. The calculator unit has to start operation about 2*N+2 cycles after the data bank unit, so the control unit has to generate their valid_in signals accordingly. The 'boundaries' and 'color_case' signals are also generated here. Lastly, this unit is responsible for generating the 'image_finished' output, which signals that image proccessing has been completed.

Top level Debayering Filter architecture:
The only thing needed in the top level filter block is to connect all the units together and simply delay a couple of signals, so that all the units are synchronized.
